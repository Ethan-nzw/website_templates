# Fix 404 on GitHub Pages

If **https://ethan-nzw.github.io/SofeagcGrasp/** shows **404**, do this:

---

## 1. Set GitHub Pages to use branch **master**

Your site was pushed to branch **master**. GitHub often defaults to **main**, which has no files → 404.

1. Open your repo: **https://github.com/Ethan-nzw/SofeagcGrasp**
2. Go to **Settings** → **Pages** (left sidebar).
3. Under **Build and deployment** → **Source**, choose **Deploy from a branch**.
4. Under **Branch**:
   - Select **master** (not `main`).
   - Folder: **/ (root)**.
5. Click **Save**.

---

## 2. Wait 1–2 minutes

GitHub rebuilds the site. After a short wait, open:

**https://ethan-nzw.github.io/SofeagcGrasp/**

You should see your site (or the template `index.html`).

---

## 3. If you want the paper page at the root URL

Right now the root URL serves **index.html** (the template). To show your **SofeagcGrasp paper** at the root:

- Either **rename**: copy `index_improved.html` over `index.html`, commit, and push.
- Or open the paper page directly: **https://ethan-nzw.github.io/SofeagcGrasp/index_improved.html**

---

## 4. Check the URL

Use exactly:

- **https://ethan-nzw.github.io/SofeagcGrasp/**  
  (username: `Ethan-nzw`, repo: `SofeagcGrasp`)

If you use a different GitHub username or repo name, the URL is:

- **https://\<username>.github.io/\<repo-name>/**  
  (no `.git`, no trailing path for the home page)
